# VED AI

## Цель проекта

Telegram-сервис и цифровой консультант по ВЭД для импортеров
автозапчастей и электроники.

## Технологии

- Python 3.13
- aiogram 3
- Telegram Bot API
- python-dotenv
- Git и GitHub

## Текущий функционал

- команда /start;
- главное меню;
- сценарий «Автозапчасти»;
- FSM-анкета;
- сбор страны, типа получателя и товара;
- итоговая карточка заявки.

## Архитектурные правила

- main.py только запускает приложение;
- handlers отвечают за Telegram-диалоги;
- keyboards содержат клавиатуры;
- states содержат состояния FSM;
- services содержат бизнес-логику и интеграции;
- config содержит настройки;
- prompts содержит инструкции для LLM;
- секреты хранятся только в .env;
- .env никогда не добавлять в Git.

## Правила разработки

- двигаться небольшими шагами;
- не усложнять архитектуру заранее;
- перед изменениями объяснять план;
- не менять несколько независимых частей одновременно;
- после изменения запускать проверку;
- не выполнять git commit и git push без прямого разрешения;
- не удалять файлы без подтверждения пользователя.

## Поведение AI-консультанта

- не гарантировать результат;
- не придумывать нормы законодательства;
- не делать окончательных выводов без достаточных данных;
- запрашивать документы и характеристики товара;
- предлагать несколько сценариев;
- объяснять риски через деньги и сроки;
- делить риски на красные, желтые и зеленые;
- явно отделять предположение от подтвержденного вывода.
